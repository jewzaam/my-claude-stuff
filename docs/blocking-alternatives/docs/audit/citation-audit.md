## Key Findings

1. **CVE number accuracy is poor.** Two citations contain CVE numbers that do not appear to exist (CVE-2025-907418 for Docker, CVE-2026-23268/23269 for CrackArmor). The research documents themselves flagged these with quality caveats, which was appropriate.

2. **Quantitative performance claims are generally accurate.** Firecracker specs (125ms, <5 MiB, 150/sec), gVisor overhead (2.2x-216x), SELinux improvements (12x policy load), AppArmor benchmarks (~5%), LlamaFirewall metrics (96% precision, 79% recall, >90% ASR reduction) -- all verified against original sources.

3. **Direct quotes are faithfully represented.** The Trail of Bits "hooks are not a security boundary" quote, the ona.com sandbox escape description, and the Backslash Security Cursor denylist assessment all match source material.

4. **Quality caveats in citations.md are well-calibrated.** Every citation flagged with "Note:" in the original citations.md had a legitimate concern: star counts were unconfirmed, CVE numbers were suspect, benchmark claims were skeptical, and first-party claims were unverified. The research team's self-assessment of data quality was honest.

5. **URL stability is an issue.** Several Anthropic documentation URLs have moved from `docs.anthropic.com` to `code.claude.com` since the research was conducted. The content remains accessible but at different URLs.
